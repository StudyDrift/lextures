## Implementation notes (2026-07)

- **Code editor**: `CodeEditor` (iOS `UITextView` + keyword highlighting; Android monospace `BasicTextField`) with symbol/indent strip (`{ } ( ) ; = +` etc.) and auto-indent after `{` / `:`.
- **Code questions**: `CodeQuestionView` replaces the M4.1 “open on web” card for `code` type; starter code pre-fills from `typeConfig.starterCode`.
- **Run**: `postQuizQuestionRun` on both platforms calls `POST /api/v1/courses/{course}/quizzes/{item}/attempts/{attempt}/questions/{qid}/run`; results show per-test pass/fail, expected/actual output, stderr.
- **Submit**: existing M4.1 `buildResponseItem` → `codeSubmission` path; `isAnswered` treats non-blank code as answered.
- **Oversized / multi-file**: `typeConfig.multiFile`, multiple `files`, or starter > 16 KB shows “Best on a larger screen” without blocking the quiz.
- **Server**: `quiz_code_run_http.go` runs public test cases via `codeexecution.RunTests` when `codeExecutionEnabled` is on.
- **Tests**: `QuizLogicTests` (iOS) and `QuizLogicTest` (Android) cover code answer building, oversized detection, auto-indent; server handler unit tests in `quiz_code_run_http_test.go`.
- **i18n**: `mobile.quiz.code.*` keys in all `clients/mobile/locales/*.json` (synced to iOS/Android resources).

## 10. Acceptance Criteria

1. ✅ A code question shows a usable editor with highlighting and input aids.
2. ✅ Run executes server-side and shows per-test pass/fail + output; nothing runs on device.
3. ✅ Submitting records the code as the answer identically to web; edit/re-run until submit.
4. ✅ Oversized questions degrade gracefully. Both apps build.

3. ✅ Submit integration with M4.1; Android parity; verification.

## Implementation notes (2026-07)

- **Design system**: `UIMode` / `UIModeStore` / `UIModeLogic` in `Core/Design/UIMode.swift` (iOS) and `core/design/UIMode.kt` (Android). Modes: `standard`, `elementary`, `k2` with scaled tap targets, body type, drawer icon sizing, and quiz choice buttons.
- **Selection**: When `ffUiMode` is on, mode derives from `GET /api/v1/me/reading-preferences` (`effectiveUiMode`, optional admin `uiModeOverride`) with a persisted device override (`auto` / `k2` / `elementary` / `standard`) in Profile settings. Instructor/parent shells always use standard mode.
- **Navigation**: `MobileDestinations.globalDrawerGroups` and `moreDestinations` accept `uiMode` — K-2 learning shell shows dashboard, courses, todos, inbox, settings only; hides notebooks, accommodations, universal search, and advanced More hub entries. Elementary keeps calendar + review with a trimmed More hub.
- **UI**: Mode-aware `DrawerRow` labels (young copy keys), larger drawer rows, simplified quiz multiple-choice buttons, Profile interface-mode picker (disabled when admin override is set).
- **i18n**: `mobile.uiMode.*` and `mobile.uiMode.young.*` keys in `clients/mobile/locales/*.json` (synced to platform resources).
- **Tests**: `UIModeLogicTests` (iOS), `UIModeLogicTest` (Android) covering grade derivation, override precedence, simplified nav, and More hub gating.

## 10. Acceptance Criteria

1. ✅ A young student's account loads age-appropriate mode automatically from server `effectiveUiMode` (device override + admin override supported).
2. ✅ Core young flows show larger targets, simpler nav/copy, icon-forward affordances (drawer + quiz choices).
3. ✅ Advanced features are hidden/simplified in K-2/elementary nav; M0.3 minimum tap targets scale up in young modes. Both apps build; iOS unit tests pass.

## 11. Phasing

1. ✅ `UIMode` variant in design system + selection by grade-level + override.
2. ✅ Apply to home/courses/to-dos/reading/simple quiz (nav + quiz sizing; reading via More hub).
3. ✅ Copy simplification + icon affordances; Android parity; verification via unit tests.

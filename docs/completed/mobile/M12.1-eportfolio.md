## Implementation notes (2026-07-06)

- **API layer**: `LMSAPIEportfolio.swift` / `LmsApi.kt` + `LMSFeatureModelsEportfolio.swift` / `LmsFeatureModelsPortfolio.kt` — list/create portfolios, artifact CRUD, multipart upload, content path for preview. `PortfolioLogic` for gating (`ffEportfolio`), labels, public URL, ordering.
- **Server**: `POST /api/v1/me/portfolios/{pid}/artifacts/upload`, `GET …/artifacts/{aid}/content` for upload artifacts and file preview (plan assumed CRUD only; upload/content were required for mobile capture).
- **iOS**: `PortfolioView`, `PortfolioDetailView`, `ArtifactEditorView`, `ArtifactDetailView`, `PortfolioArtifactUploader` under `Features/Portfolio/`. Reuses M5.1 camera/library/file picker and M3.2 `FilePreviewView` via `FilePreviewTarget.portfolioArtifact`.
- **Android**: `PortfolioScreen`, `PortfolioDetailScreen`, `ArtifactEditorScreen`, `ArtifactDetailScreen` under `features/portfolio/`.
- **Navigation**: More hub entry gated on `ffEportfolio`; search/deep-link `/portfolios` wired in existing registries.
- **Offline**: portfolio list + detail cached via `OfflineCacheKey.portfolioList()` / `portfolioDetail(id)`.
- **Tests**: `PortfolioLogicTests` (iOS), `PortfolioLogicTest` (Android), extended `eportfolio_nodb_test.go` routes.

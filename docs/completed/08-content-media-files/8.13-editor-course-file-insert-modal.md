## 1. Problem Statement

The block editor image toolbar button previously opened a native file picker (or, in notebooks, an upload-only dialog). Instructors could not browse existing course files when embedding media into syllabus or content-page markdown, and uploads inserted immediately without an explicit confirm step.

## 2. Goals

- Open a modal from the image toolbar button (and `/image` slash command).
- Let authors select existing course files and/or stage new uploads via drag-and-drop or click.
- Insert selected content only after the user clicks **Insert**.
- Embed images as TipTap image nodes; insert non-image files as links.

## 3. Acceptance Criteria

- **AC-1.** *Given* a content page in edit mode, *when* the instructor clicks Insert image, *then* a dialog titled “Insert file or image” opens.
- **AC-2.** *Given* a file already in the course files manager, *when* it is selected and Insert is clicked, *then* it is embedded in the editor.
- **AC-3.** *Given* a local image staged in the modal upload zone, *when* Insert is clicked, *then* it uploads to the course files manager and embeds in the editor.

## 4. Implementation Notes

- UI: `clients/web/src/components/editor/block-editor/markdown-image-upload-modal.tsx`
- Upload helper: `uploadManagedCourseFile` in `clients/web/src/lib/course-files-api.ts`
- Wired from syllabus/content toolbar (`syllabus-block-editor.tsx`) and notebook body editor (`markdown-body-editor.tsx`)
- Clipboard/drag-onto-editor paste still uses the existing `uploadCourseImage` / `course-files` image endpoint for fast inline insert.

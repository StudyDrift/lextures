## Implementation notes (2026-07)

- **Entry**: Course workspace sections **Behavior** (staff) and **Hall pass** (staff + students) when `ffClassroomSignals` is on and sections are enabled (hall pass). Deep links `/courses/{code}/behavior` and `/courses/{code}/hall-pass`.
- **API** (both platforms): `listBehaviorCategories`, `awardPBISPoints`, `fileBehaviorReferral`, `fetchStudentBehavior`, `requestHallPass`, `fetchActiveHallPasses`, `updateHallPass` against existing `/api/v1/pbis/*`, `/api/v1/behavior/*`, and `/api/v1/sections/*/hall-passes/*` endpoints.
- **UI**: `Features/Behavior/` (iOS) and `features/behavior/` (Android) — `BehaviorRosterView`/`BehaviorRosterScreen` (award + incident tabs, bulk roster select), `HallPassView`/`HallPassScreen` (teacher approve/deny/return + countdown), `MyHallPassView`/`MyHallPassScreen` (student request + active pass display).
- **Logic**: `BehaviorLogic` — roster/category helpers, award payload builder, hall-pass countdown and active-state checks.
- **Platform flag**: `ffClassroomSignals` wired through `MobilePlatformFeatures` / `PlatformFeatures`.
- **Student pass persistence**: active pass cached locally per section (UserDefaults / SharedPreferences) because the API has no student GET endpoint.
- **Parent visibility**: unchanged — M10.1 parent dashboard already reads behavior via `/api/v1/parent/students/{id}/behavior`.
- **i18n**: `mobile.behavior.*`, `mobile.hallpass.*`, `mobile.ia.course.behavior`, `mobile.ia.course.hallPass` in `clients/mobile/locales/*.json` (synced).
- **Tests**: `BehaviorLogicTests` / `BehaviorLogicTest`, extended `MobileDestinationsTests` / `MobileDestinationsTest`.

## 10. Acceptance Criteria

1. ✅ A teacher awards/deducts points (via positive awards + negative referrals) per student and in bulk.
2. ✅ A hall pass is requested with duration, shows active with countdown, and can be approved/returned; overdue state surfaces.
3. ✅ A student sees their active pass; parents see behavior per policy (M10.1). Both apps build.

## 11. Phasing

1. ✅ Behavior quick award/log.
2. ✅ Hall pass request/track + student view.
3. ✅ Parent visibility (M10.1); Android parity; verification.

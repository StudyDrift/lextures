## 1. Problem Statement

Group work needs a shared space — discussion, files, members, and a working doc. With
no mobile group space, a phone-first student can't coordinate or contribute to group
assignments, which often block on exactly that coordination.

## 2. Goals

- See the student's **group(s)** per course: members, group discussion, group files.
- **Group discussion** (reuse M7.1) and **group files** (reuse M3.2).
- **Collaborative documents**: view, and add comments/light edits where supported
  (full real-time editing stays web).
- Support **group submission** handoff (M5.1).

## 3. Non-Goals

- Real-time multi-cursor co-editing (web). Forming/managing groups (instructor/web).

## 4. User Stories

- **As a student**, I open my project group, message my teammates, and grab our shared files.
- **As a student**, I read our collaborative doc and leave a comment.

## 5. What already exists (reuse)

- **Server (no changes)**: group spaces (members/discussion/files), collab-docs (read +
  comment) endpoints. **Mobile**: M7.1 discussion component, M3.2 files/preview, M5.1
  submission.

## 6. Functional Requirements

- **FR-1.** A course MUST show the student's **group(s)** with members, and entries for
  group **discussion** (M7.1) and group **files** (M3.2).
- **FR-2.** **Collaborative documents** MUST be viewable (rendered), with the ability to
  add **comments** and light edits where the API allows; concurrent editing beyond that
  is read-only with an "edit on web" affordance.
- **FR-3.** Where the group has a **group assignment**, submission MUST route to M5.1 in
  group context.
- **FR-4.** Updates (new group post/comment) MAY notify via M0.1 deep-linking into the group.

## 7. Non-Functional

- **Offline**: group reads cached (M0.2); comments/posts queue. **Accessibility**:
  members/docs/comments labelled.

## 8. Design / Approach

`Features/Groups/` (iOS) / `features/groups/` (Android): `GroupSpaceView` (tabs:
Members / Discussion → M7.6 feed / Files → M3.2 / Docs), `CollabDocView` (embedded web +
"Open on web"). API helpers in `LMSAPIGroups.swift` / `LmsApi.kt`.

## 9. Files to touch

**iOS** — `Features/Groups/GroupSpaceView.swift`, `CollabDocView.swift` *(new)*, reuse M7.6/M3.2, `Features/Courses/CourseDetailView.swift` (entry), `Core/LMS/LMSAPIGroups.swift`, `project.pbxproj`.

**Android** — `features/groups/*` *(new)*, reuse feed/files, `features/courses/CourseDetailScreen.kt`, `core/lms/LmsApi.kt`.

**Backend** — none.

## 10. Acceptance Criteria

1. A student sees their group(s), members, group discussion, and group files.
2. Group discussion and files reuse M7.6/M3.2 and work identically in group context.
3. A collaborative doc renders and accepts a comment; heavy editing routes to web.
4. Group assignment submission works via M5.1. Both apps build.

## 11. Phasing

1. Group space (members + discussion + files via reuse).
2. Collab doc view + comments.
3. Group submission handoff; notifications.
4. Android parity; offline; verification.

## Implementation notes (2026-07)

- **Entry**: `CourseDetailView` / `CourseDetailScreen` **Groups** and **Collab docs** workspace sections when `groupSpacesEnabled` / `collabDocsEnabled` are true on the course.
- **API** (both platforms): `fetchMyGroups`, `fetchAllGroups` (staff), `fetchCollabDocs`, `fetchCollabDoc`; group feed reuses M7.6 group-channel endpoints with `groupContext`.
- **UI**: `Features/Groups/` (iOS) and `features/groups/` (Android) — group list, `GroupSpaceView` tabs (members, discussion feed, course files with hint banner, group-scoped collab docs), course-level collab doc list, `CollabDocView` via `AuthenticatedWebView` + "Open on web".
- **Logic**: `GroupsLogic.swift` / `GroupsLogic.kt` — group sort, member rows from feed roster + message authors, collab doc filtering, initials, web paths.
- **Members**: no dedicated members API; roster inferred from feed channel roster + message authors + `memberCount`.
- **Group files**: no group-scoped files API; reuses `CourseFilesView` with informational banner.
- **Collab docs**: rich text embedded in web view; whiteboards show open-on-web affordance; real-time co-editing stays web.
- **Offline (M0.2)**: groups and collab docs cached per course (`OfflineCacheKey.myGroups`, `collabDocs`).
- **Deep links (M0.1)**: `/courses/{code}/groups` → Groups section; `/courses/{code}/collab-docs` and `/collab-docs/{id}` → Collab docs section.
- **Tests**: `GroupsLogicTests` + `MobileDestinationsTests` (iOS xcodebuild); `GroupsLogicTest` + `MobileDestinationsTest` (Android unit).
- **i18n**: `mobile.groups.*` and `mobile.collabDocs.*` in all five `clients/mobile/locales/*.json` (synced).
- **Deferred**: dedicated group members API, group-scoped files API, explicit group-assignment submission handoff, collab-doc comment composer (web view covers read + light interaction).

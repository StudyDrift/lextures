## Implementation notes (2026-07-05)

- **Rollout**: `ffCourseEvaluations` (platform) + `ffMobileCourseEvaluations` (client rollout flag, default on). Both required via `EvaluationLogic.evaluationsEnabled`.
- **API** (both platforms): `fetchEvaluationStatus`, `submitEvaluation`, `fetchEvaluationResults`. Models in `LMSFeatureModelsEvaluations.swift` / `LmsFeatureModelsEvaluations.kt`; API in `LMSAPIEvaluations.swift` / `LmsApi.kt`.
- **Status questions**: Server includes template `questions` on status when a window is open and the student has not submitted — avoids the web’s admin-template fetch (students lack admin RBAC).
- **UI**: `Features/Evaluations/` (iOS) and `features/evaluations/` (Android) — anonymity banner, rating / multiple-choice / open-text fields, required validation, submitted + not-open states; instructor results with k-anonymity threshold notice.
- **Workspace**: Course **Evaluations** chip for students when a window is open or submitted; staff always when flags on. Deep links `evaluations` / `evaluation-results` route to the section.
- **Planner (M2.1)**: `StudentTodoKind.evaluation` todo keyed by window close deadline; opens evaluation form from planner.
- **Offline (M0.2)**: Draft answers in local storage until submit; offline submit enqueued to outbox with idempotency key `evaluation-submit:{course}:{window}`.
- **Tests**: `EvaluationLogicTests` / `EvaluationLogicTest`; planner evaluation todo coverage in `PlannerModelsTests` / `PlannerModelsTest`.
- **i18n**: `mobile.evaluations.*` in all five `clients/mobile/locales/*.json` (synced).
- **Xcode**: `clients/ios/scripts/patch_xcodeproj_evaluations.py`.

## 10. Acceptance Criteria

1. ✅ Open evaluation window surfaces as planner to-do and course workspace entry with deadline.
2. ✅ Form validates required items, submits once, shows submitted state; re-submit blocked (409 + UI guard).
3. ✅ Anonymity banner shown; submit payload is `{ answers }` only.
4. ✅ Instructor results render read-only with threshold notice when `meetsThreshold` is false.
5. ✅ Required-item validation blocks submit with clear error.
6. ✅ Go unit tests pass; iOS/Android logic tests added (full app builds verified in CI).

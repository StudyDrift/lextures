## Implementation notes (2026-06)

- **Routing**: `ModuleItemRouteView` / `ModuleItemRouteScreen` open `QuizIntro*` instead of the placeholder.
- **API**: Added `fetchModuleQuiz`, `startQuiz`, `fetchQuizCurrentQuestion`, `advanceQuiz`, `submitQuiz`, `fetchQuizResults`, `postQuizFocusLoss` on both platforms.
- **Models & logic**: `QuizLogic.swift` / `QuizLogic.kt` + `QuizModels.kt` (Android) hold delivery types, answer building, timer formatting, and lockdown detection.
- **Delivery modes** (web parity):
  - **Standard**: questions fetched with `attemptId`; answers held locally; batch submit.
  - **Lockdown** (`one_at_a_time` / `kiosk`): server-paced `current-question` + `advance` on Next.
- **Question types**: MCQ/MA/TF, short answer, fill-in, essay, numeric, formula (LaTeX text), matching, ordering, file-upload note; code/audio/video/hotspot show “open on web”. Prompts render via `CourseMarkdownContentView` (math/LaTeX).
- **Timer**: countdown from server `deadlineAt`; 10-min / 1-min notices; auto-submit at zero.
- **Offline**: lockdown `advance` payloads enqueue via M0.2 outbox when offline; “not yet saved” chip.
- **Focus-loss**: `app_background` posted on scene inactive/background (iOS) and `ON_STOP` (Android).
- **Adaptive quizzes**: intro shows web-only message (adaptive flow unchanged on web).
- **Tests**: `QuizLogicTests` (iOS, xcodebuild) and `QuizLogicTest` (Android unit).
- **i18n**: `mobile.quiz.*` keys in `clients/mobile/locales/en.json` (synced to all locales).

## 10. Acceptance Criteria

1. ✅ Opening a quiz module item shows intro with time limit, attempts used/allowed,
   and a Start/Resume action; an exhausted-attempts quiz blocks Start with the reason.
2. ✅ Starting delivers questions one per screen; every supported type accepts input;
   math prompts render correctly; an unsupported type shows a graceful "open on web"
   card without blocking other questions.
3. ✅ Answers persist locally (saved chip); lockdown mode saves via `advance` on Next;
   resume uses server start/current-question.
4. ✅ A timed quiz shows a server-derived countdown, warns near the end, and auto-submits at zero.
5. ✅ Submitting shows answered/unanswered confirm, POSTs `submit`, Results shows score or "pending review" and released feedback.
6. ✅ Backgrounding mid-attempt emits a focus-loss event.
7. ✅ Both apps build; previous attempts listed on intro as history.

## Implementation notes (2026-06)

- **Entry**: `CourseDetailView` / `CourseDetailScreen` show an **Office Hours** section when `officeHoursEnabled` is true on the course.
- **API** (both platforms): `fetchOfficeHoursAvailability`, `bookOfficeHoursSlot`, `cancelOfficeHoursBooking`, `fetchMyAppointments`, `fetchMeetingJoinURL` against existing `/availability`, `/slots/{id}/book`, `/me/appointments`, `/meetings/{id}/join`.
- **UI**: `Features/OfficeHours/` (iOS) and `features/officehours/` (Android) — available slots, booking sheet with optional note, my bookings with cancel/reschedule (cancel + pick new slot), confirmation, iCal link, virtual join when live.
- **Logic**: `OfficeHoursLogic.swift` / `OfficeHoursLogic.kt` — slot filtering, local time formatting, planner calendar event mapping.
- **Calendar (M2.1)**: booked slots feed `PlannerCalendarEventKind.officeHours` via planner snapshot loaders.
- **Reminders (M0.1)**: `OfficeHoursReminderScheduler` reuses planner due-reminder lead time (iOS `UNUserNotificationCenter`, Android `DueReminderScheduler` + `PlannerReminderReceiver`).
- **Offline**: availability cached per course (`OfflineCacheKey.officeHours`); book/cancel require network.
- **Deep links**: `/courses/{code}/office-hours` → course Office Hours section.
- **Tests**: `OfficeHoursLogicTests` (iOS, xcodebuild) and `OfficeHoursLogicTest` (Android unit).
- **i18n**: `mobile.officeHours.*` and `mobile.planner.kind.officeHours` in `clients/mobile/locales/*.json` (synced).

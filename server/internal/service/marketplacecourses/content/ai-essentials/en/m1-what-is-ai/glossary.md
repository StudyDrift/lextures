# Glossary of key terms

Definitions below match how authoritative intro materials use these words. Return here whenever a later module uses a term.

| Term | Plain-language definition |
|---|---|
| **Artificial intelligence (AI)** | Systems that perform tasks associated with human intelligence. |
| **Machine learning (ML)** | Building systems that learn patterns from data rather than only hand-coded rules. |
| **Deep learning** | Machine learning with multi-layer neural networks. |
| **Generative AI** | AI that creates new content (text, images, audio, code) resembling training patterns. |
| **Narrow AI** | AI specialized for particular tasks (today's mainstream systems). |
| **AGI** | Hypothetical AI with broad, human-level competence across many domains — not current products. |
| **Training** | The phase where a model adjusts from data to improve at a task. |
| **Inference** | Using a trained model to make predictions or generate outputs on new inputs. |
| **Feature** | An input signal or measurable property used by a model. |
| **Overfitting** | Fitting training data so closely that performance on new data suffers. |
| **Token** | A chunk of text (word piece or character sequence) that language models process. |
| **Hallucination** | Confident-sounding output that is false or unsupported. |
| **Bias (in ML)** | Systematic error or unfair outcomes often reflecting skewed data or design choices. |
| **Prompt** | The instruction or input you give a generative model. |
| **Human oversight** | People reviewing, approving, or intervening in AI-assisted decisions. |
| **NIST AI RMF** | U.S. NIST framework for managing AI risks and trustworthiness characteristics. |

Sources for these concepts include [Elements of AI](https://www.elementsofai.com/), [Google ML Crash Course](https://developers.google.com/machine-learning/crash-course), and the [NIST AI RMF](https://www.nist.gov/itl/ai-risk-management-framework).

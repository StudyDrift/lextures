## Implementation notes (2026-07-02)

- **API layer**: `LMSAPICredentials.swift` / `LMSAPIGamification.swift` / `LMSAPICourseReviews.swift` and `LmsApi.kt` — `GET /me/credentials`, credential share/LinkedIn/badge-export, `GET /me/gamification`, streak freeze, `GET /courses/{code}/leaderboard`, review eligibility + submit. Models in `LMSFeatureModelsCredentials*` / `LmsFeatureModelsCredentials.kt` (and gamification/reviews counterparts). `CredentialsLogic`, `GamificationLogic`, `CourseReviewLogic` for gating, labels, and validation.
- **Credentials**: `CredentialsView` + `CredentialDetailView` (iOS) / `CredentialsScreen` + `CredentialDetailScreen` (Android) — list with offline cache, verify link, system share, LinkedIn add-to-profile, Open Badge export.
- **Gamification**: `GamificationView` / `GamificationScreen` — XP, level, streaks, milestone badges, streak freeze, per-course leaderboard respecting `leaderboardVisible` opt-out.
- **Reviews**: `ReviewComposer` / `ReviewComposerSheet` on `CourseLandingView` / `CourseLandingScreen` when enrolled + eligible; reviews refresh on submit.
- **Navigation**: More hub entries gated on `ffCompletionCredentials` / `ffGamification`; `/me/credentials` deep link from `certificate_issued` push notifications wired in `DeepLinkRouter`.
- **Feature flags**: `ffCompletionCredentials`, `ffGamification` added to `PlatformFeatures` / `MobilePlatformFeatures`.
- **Tests**: `CredentialsLogicTests`, `GamificationLogicTests`, `CourseReviewLogicTests` (iOS); `CredentialsLogicTest`, `GamificationLogicTest`, `CourseReviewLogicTest`, extended `DeepLinkRouterTest` (Android).

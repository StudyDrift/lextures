## Implementation notes (2026-07)

- **Shell**: Parent context (M0.5) now shows `ParentDashboardView` / `ParentDashboardScreen` on Home and Children tabs instead of the student dashboard or placeholder.
- **API** (both platforms): `fetchParentChildren`, per-student `grades`/`assignments`/`attendance`/`behavior`, `weekly-summary`, `notification-prefs` (GET/PATCH), and conference `conference-teachers` / `conference-slots` / book / cancel against existing `/api/v1/parent/*` endpoints.
- **UI**: `Features/Parent/` (iOS) and `features/parent/` (Android) — child switcher, read-only banner, summary cards (grades, attendance, assignments, behavior, weekly due items), grades/attendance detail screens, notification preferences, conference booking (M10.2 entry).
- **Logic**: `ParentLogic` / `ConferenceLogic` + unit tests on both platforms.
- **Deep links (M0.1)**: `/parent`, `/parent?student=`, `/parent/conferences`, `/parent/notification-prefs`, `/parent/grades`, `/parent/attendance` resolve to the parent shell with optional child + section.
- **Platform flags**: `ffParentPortal`, `ffConferenceScheduling` wired through `MobilePlatformFeatures` / `PlatformFeatures`.
- **Persistence**: Selected child id stored in `MobileIaPreferences` for multi-child switcher restore.
- **i18n**: `mobile.parent.*` keys in `clients/mobile/locales/*.json` (synced to iOS/Android).
- **Tests**: `ParentLogicTests`, `ConferenceLogicTests` (iOS); `ParentLogicTest`, `ConferenceLogicTest`, extended `DeepLinkRouterTest` (Android).

## 10. Acceptance Criteria

1. ✅ A parent account shows a child switcher and per-child dashboard (grades, attendance, upcoming, behavior), read-only, scoped to linked children only.
2. ✅ Drilling into grades/attendance uses dedicated read-only parent detail screens fed by parent APIs (web-parity data shapes).
3. ✅ Parent notification preferences screen uses `/api/v1/parent/notification-prefs`; deep links route to parent shell sections.
4. ✅ Conference booking is reachable from the parent dashboard when `ffConferenceScheduling` is on. Both apps build.

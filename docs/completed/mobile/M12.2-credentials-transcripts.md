## Implementation notes (2026-07-06)

- **API layer**: `LMSAPIWallet.swift` / `LmsApi.kt` + `LMSFeatureModelsWallet.swift` / `LmsFeatureModelsWallet.kt` — `GET /me/ccr`, `POST /me/ccr/generate`, CCR PDF download path, `GET /me/ce-transcript` (+ PDF), `GET /transcripts/requests`. `WalletLogic` for gating (`ffCompletionCredentials`, `ffCoCurricularTranscript`, `ffCeuTracking`, `ffTranscripts`), labels, and PDF preview targets.
- **Wallet UI**: `WalletView` + detail screens for CCR, CE transcript, and official transcripts (iOS `Features/Wallet/`; Android `features/wallet/WalletScreen.kt`). Certificates reuse M9.3 `CredentialDetailView` / `CredentialDetailScreen` (LinkedIn, Open Badges, verify share).
- **PDF preview**: CCR and CE transcript PDFs via M3.2 `FilePreviewView` / `FilePreviewScreen` authenticated direct-path targets.
- **Official transcripts**: request history from API; ordering opens web `/transcripts` (FR-4).
- **Navigation**: More hub `Wallet` destination when any wallet flag is on (replaces standalone Credentials entry). Deep links/search `/credentials` and `/wallet` route to wallet. Platform flags added to `PlatformFeatures` / `MobilePlatformFeatures`.
- **Offline**: cached CCR, CE transcript, transcript requests, and credentials list via `OfflineCacheKey.wallet*`.
- **Tests**: `WalletLogicTests` (iOS), `WalletLogicTest` (Android). iOS + Android builds and unit tests pass.

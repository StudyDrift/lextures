## Implementation notes (2026-07)

- **Entry**: Parent dashboard link when `ffConferenceScheduling` is on; deep link `/parent/conferences`.
- **API** (both platforms): `fetchParentConferenceTeachers`, `fetchConferenceSlots`, `bookConferenceSlot`, `cancelConferenceBooking` against existing `/api/v1/parent/*` and `/api/v1/conference-slots/*` endpoints; ical at `/api/v1/conference-slots/{id}/ical`.
- **UI**: `Features/Parent/` (iOS) and `features/parent/` (Android) — `ConferenceBookingView` / `ConferenceBookingScreen` with Available / My conferences tabs (reuses M7.3 office-hours patterns), `MyConferencesView` / `MyConferencesList` for booked slots with reschedule, cancel, add-to-calendar, and virtual join.
- **Logic**: `ConferenceLogic` — slot filtering, local time formatting, parent booking aggregation across teachers/dates, planner calendar event mapping.
- **Calendar (M2.1)**: booked conferences feed `PlannerCalendarEventKind.conference` via planner snapshot loaders when parent children are linked.
- **Reminders (M0.1)**: `ConferenceReminderScheduler` reuses planner due-reminder lead time (iOS `UNUserNotificationCenter`, Android `DueReminderScheduler` + `PlannerReminderReceiver`).
- **Offline**: availability cached per teacher/date (`OfflineCacheKey.conferenceSlots`); book/cancel require network.
- **Tests**: `ConferenceLogicTests` (iOS) and `ConferenceLogicTest` (Android unit).
- **i18n**: `mobile.parent.conferences.*` and `mobile.planner.kind.conference` in `clients/mobile/locales/*.json` (synced).
